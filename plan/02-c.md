**Why C matters**: it is Holzmann's original target language. Several Power of
Ten clauses that every other language *adapts away* apply literally here
(goto/setjmp, dynamic allocation, the real preprocessor, raw pointers), so C
brings the largest set of new language-specific rules of any addition. Expect
the rule-design half to dominate the work.

**Scope**: one comprehensive MINOR release. Grammar `tree-sitter-c`,
extensions `.c` and `.h`, line comments `//`.

**`.h` ownership decision**: register `.h` to C now. C++ (plan/03) keeps the
C++-only extensions (`.hpp` etc.); a C++ project using `.h` headers gets them
linted as C, which is the documented limitation (content sniffing is out of
scope). State this in both language pages.

## 1. Language module (`src/safelint/languages/c.py`)

- `EXTRA_NAME = "c"`, extra `c = ["tree-sitter-c>=0.23.0"]` + `[all]`.
- `LanguageDefinition(name="c", file_extensions={".c", ".h"},
  comment_node_type="comment", comment_prefix="//")`.
- Key node types (verify by probing): `function_definition` (the only
  FUNCTION_TYPES member; no closures), `call_expression`, `if_statement`,
  `for_statement`, `while_statement`, `do_statement`, `switch_statement` /
  `case_statement`, `goto_statement`, `labeled_statement`,
  `preproc_def` / `preproc_function_def` / `preproc_if` / `preproc_ifdef` /
  `preproc_include`, `pointer_declarator`, `declaration`,
  `conditional_expression`.

## 2. Per-rule portability audit

### Ports cleanly

| Rule | C shape / notes |
|---|---|
| SAFE101-104 | `function_definition`; complexity counts `if` / `for` / `while` / `do` / `case_statement` / ternary / `&&` `\|\|` via operator filter. |
| SAFE105 `no_recursion` | Bare identifier calls only (no methods, no receivers) - the simplest port of all. |
| SAFE302 `global_mutation` | Declaration-site like Java/Go: file-scope `declaration`s that are not `const`-qualified and not function prototypes. `static` file-scope variables still count (shared within the translation unit). |
| SAFE303 / SAFE304 | `io_functions_c`: `printf`, `fprintf`, `puts`, `putchar`, `fopen`, `fread`, `fwrite`, `fgets`, `scanf`, `getchar`, `read`, `write`, `recv`, `send`, `system`. |
| SAFE309 `dynamic_code_execution` | No eval; the C surface is dynamic loading: `dynamic_exec_calls_c = ["dlopen", "dlsym"]`. |
| SAFE501 `unbounded_loops` | `while (1)` / `while (true)` / `for (;;)` (empty condition) without an exiting break. No labelled break in C; `goto` exits are the new wrinkle - a `goto` jumping out of the loop counts as an exit (resolve the label position). |
| SAFE601 `missing_assertions` | C's `assert(...)` from `<assert.h>` parses as a plain `call_expression` - register with `assertion_calls_c = ["assert"]`. The paper's own example language finally gets its literal rule 5. |
| SAFE603 `blanket_suppression` | Bare `// NOLINT` / `// NOLINTNEXTLINE` (no `(category)` suffix) from clang-tidy. Scoped `NOLINT(bugprone-*)` clean. |
| SAFE701 / SAFE702 | Weak conventions; register with defaults (`test_dirs = ["tests", "test"]`, pair `foo_test.c` / `test_foo.c`) and document that C projects usually override. |
| SAFE801 `tainted_sink` | New `analysis/dataflow_c.py`. `sinks_c`: `system`, `popen`, `execl`, `execlp`, `execv`, `execvp`, `sprintf`, `strcpy`, `strcat`, `gets`, `memcpy`. `sources_c`: `getenv`, `fgets`, `scanf`, `gets`, `read`, `recv`, `argv` (function-parameter seeding covers `argv` naturally). `sanitizers_c`: generic narrow set. |
| SAFE802 `return_value_ignored` | `flagged_calls_c`: `fclose`, `fwrite`, `fread`, `remove`, `rename`, `fflush`, `setvbuf`, `snprintf` (truncation), `malloc`-family is covered by SAFE310 instead. The paper's rule 7 names `printf`/`close` as the canonical violations; an explicit `(void)f()` cast does NOT fire (the cast wraps the call so it is no longer a bare expression-statement call - verify the node shape and add a test). |

### Deliberately skipped

| Rule | Rationale |
|---|---|
| SAFE201 / SAFE202 / SAFE203 | No try/catch. Error-handling discipline is SAFE802's return-code checking. |
| SAFE301 | No `global` keyword; file-scope state covered by the SAFE302 port. |
| SAFE305 | No hoisting distinction. |
| SAFE401 `resource_lifecycle` | C cleanup idioms (`goto err` chains, early returns with explicit `fclose`/`free`) need flow analysis the rule does not do; allocation discipline is the new SAFE310's job and release-pairing is left to dedicated leak checkers. Documented gap. |
| SAFE803 `null_dereference` | Chained deref straight off a call (`*fopen(...)`) is rare C style; nullable-return tracking (`malloc`, `fopen`, `getenv`) without flow analysis would be noise. Documented gap; revisit with demand. |

### New C-only rules (the Holzmann homecoming; verify codes free, all opt-in unless noted)

| Proposed | Name | Band | Behaviour |
|---|---|---|---|
| SAFE106 | `nonlocal_jumps` | 1xx (rule 1, literal) | Flags every `goto_statement` and every call to `setjmp` / `longjmp` / `sigsetjmp` / `siglongjmp`. **Maintainer decision point**: the paper bans goto outright, but `goto err` cleanup is pervasive idiomatic C - proposal is `enabled = true, severity = "warning"` so it surfaces but does not block, with `# nosafe`-style `// nosafe: SAFE106` for sanctioned cleanup chains. Ship disabled instead if the noise is unacceptable on real codebases (trial on a vendored C project first). |
| SAFE310 | `dynamic_allocation` | 3xx (rule 3, literal) | Flags calls to `malloc`, `calloc`, `realloc`, `aligned_alloc`, `free`, `strdup`. Disabled by default (embedded / safety-critical projects opt in); configurable `allocation_calls_c`. |
| SAFE311 | `complex_macro` | 3xx (rule 8: macros must be simple, complete syntactic units) | Flags `preproc_function_def` whose body contains token pasting (`##`) or variadic `__VA_ARGS__`, and object-like macros whose replacement is not a complete syntactic unit (heuristic: unbalanced braces/parens in the replacement text). Disabled by default. |
| SAFE312 | `conditional_compilation` | 3xx (rule 8: each directive doubles the test matrix) | Flags every `preproc_if` / `preproc_ifdef` / `preproc_ifndef` beyond the include-guard pattern (an `#ifndef X` + `#define X` pair opening a `.h` file is exempt). Disabled by default; the message cites the 2^n versions rationale. |
| SAFE313 | `restricted_pointers` | 3xx (rule 9, literal) | Flags declarators with more than one pointer level (`int **p`) and function-pointer declarators. Disabled by default. |

These five close the four Power-of-Ten clauses that no existing language could
express literally. Update `docs/power-of-ten.md` rows 1, 3, 8, 9 to point at
them for C (the "adapted-away" sections gain "except on C, where the literal
rule exists as SAFE3xx" notes).

## 4. Tests
