# Go language support - implementation spec

**Status**: not started. Read `plan/README.md` first (sequence, non-negotiables,
validation gate), then `docs/contributing/adding-a-language.md` and the
`add-language-support` skill. This spec contains only the Go-specific design
decisions; the generic mechanics live in those references.

**Scope**: one comprehensive MINOR release. Grammar `tree-sitter-go`,
extensions `.go`, line comments `//` (the established directive convention;
block comments are a documented non-feature, same as JS / Java / Rust).

---

## 1. Language module (`src/safelint/languages/go.py`)

- `EXTRA_NAME = "go"`, extra `go = ["tree-sitter-go>=0.23.0"]` in
  pyproject + fold into `[all]`.
- `LanguageDefinition(name="go", file_extensions={".go"},
  comment_node_type="comment", comment_prefix="//")`.
- Key node types to export (verify by probing): `function_declaration`,
  `method_declaration`, `func_literal` (closures) - these three form
  `FUNCTION_TYPES`. Control flow: `if_statement`, `for_statement` (Go's only
  loop keyword: bare `for {}`, `for cond {}`, three-clause, `for range`),
  `expression_switch_statement`, `type_switch_statement`, `select_statement`,
  `go_statement`, `defer_statement`. Calls: `call_expression` with
  `selector_expression` for `pkg.Fn` / `recv.Method`. Declarations:
  `var_declaration`, `const_declaration`, `short_var_declaration`.

## 2. Per-rule portability audit

### Ports cleanly (widen `language` tuple, add Go node tables)

| Rule | Go shape / notes |
|---|---|
| SAFE101 `function_length` | `function_declaration` / `method_declaration` / `func_literal`. |
| SAFE102 `nesting_depth` | `if` / `for` / switch forms / `select`. |
| SAFE103 `max_arguments` | `parameter_declaration` children; one declaration can bind several names (`a, b int`) - count names, not declarations. Method receiver is NOT a parameter. |
| SAFE104 `complexity` | `if` / `for` / each `expression_case` + `type_case` + `communication_case` / `&&` `\|\|` via binary_expression operator filter. |
| SAFE105 `no_recursion` | Bare `call_expression` identifier match, plus receiver-qualified self-call: in `func (s *Svc) Walk()`, `s.Walk(...)` is self-recursion - match the receiver *name* from the method's receiver field (Go's analogue of `self`/`this` is the user-chosen receiver identifier). |
| SAFE302 `global_mutation` | **Declaration-site, like Java**: package-level `var` declarations (not `const`) are Go's shared mutable state. Flag each `var_spec` name at file scope. Function-body write analysis is a non-goal (shadowing via `:=` makes it unreliable single-file). |
| SAFE303 / SAFE304 side effects | `io_functions_go`: `Print`, `Println`, `Printf`, `Fprintf` (fmt), `Open`, `Create`, `ReadFile`, `WriteFile` (os), `Get`, `Post`, `Do` (net/http), `Dial`, `Listen` (net), `Exec`, `Query` (database/sql). `call_name` resolves `fmt.Println` via the selector field. |
| SAFE309 `dynamic_code_execution` | Go has no `eval`; the analysability threat is reflection and plugin loading. `dynamic_exec_calls_go`: `Call`, `CallSlice`, `MethodByName` (reflect), `Open`, `Lookup` (plugin). Bare-receiver caveat documented like Java's `forName`. |
| SAFE401 `resource_lifecycle` | `tracked_functions_go`: `Open`, `Create` (os), `Dial`, `Listen` (net), `Open` (sql)... safe form is a `defer x.Close()` in the same function body (Go-specific detection: a `defer_statement` whose call's receiver matches the variable the acquirer was assigned to). Mirrors the Java try-with-resources strictness work. |
| SAFE501 `unbounded_loops` | Bare `for {}` (no condition) without an exiting `break` = Go's `while true`. Labelled break exists (`break outer`) - reuse the labelled-break resolution from Java/Rust. `for range` / conditioned `for` are bounded shapes. |
| SAFE603 `blanket_suppression` | Bare `//nolint` (no `:rule` suffix; golangci-lint treats it as all-linters) vs scoped `//nolint:errcheck` clean. Also bare `//lint:ignore` without a reason. Comment-node scan like Python/JS. |
| SAFE701 / SAFE702 tests | Go's convention is **sibling** `foo_test.go`, not a `tests/` dir - this is the headline Go adaptation (like Rust's inline-test bypass). Pair `foo.go` with `foo_test.go` in the same directory; `test_dirs` is not the Go idiom. |
| SAFE801 `tainted_sink` | New `analysis/dataflow_go.py` (iterative worklists). `sinks_go`: `Command`, `CommandContext` (os/exec), `Query`, `QueryRow`, `Exec` (database/sql raw SQL), `Open` (plugin). `sources_go`: `Getenv`, `Args` (os), `FormValue`, `PostFormValue`, `Query` on URL, `Header.Get`. `sanitizers_go`: narrow generic set (`sanitize`, `validate`, `escape`, `quote`) per the Java/Rust single-set rationale. |
| SAFE802 `return_value_ignored` | The flagship Go rule: ignoring `error` returns. `flagged_calls_go`: `Write`, `Close`, `Remove`, `RemoveAll`, `Rename`, `Setenv`, `Chmod`, `MkdirAll` (os / io), `Rollback`, `Commit` (sql.Tx). Bare expression-statement calls only; `_ = f()` and `x, _ := f()` are explicit, auditable discards and do NOT fire (Go's `(void)`-cast analogue, document on power-of-ten.md). |

### Deliberately skipped (record each in the language page's "not registered" list)

| Rule | Rationale |
|---|---|
| SAFE201 `bare_except` | No try/catch in Go. |
| SAFE202 / SAFE203 | No catch blocks; the spirit (silently swallowed / unlogged errors) is covered by the new Go-only SAFE209 below plus SAFE802. |
| SAFE301 `global_state` | No `global` keyword; package-level state is covered by the SAFE302 port. |
| SAFE305 `wide_scope_declaration` | No `var` hoisting; `:=` is block-scoped. |
| SAFE601 `missing_assertions` | Go has no production assertion idiom (no `assert` keyword or stdlib macro; testify is test-only). Register nothing rather than a noisy heuristic; revisit if demand appears. |
| SAFE803 `null_dereference` | No chained-nullable idiom to anchor on (nil-pointer analysis needs type information; map reads return zero values). Documented gap. |

### New Go-only rules (band by category; verify codes are still free)

| Proposed | Name | Band rationale | Behaviour |
|---|---|---|---|
| SAFE209 | `empty_error_check` | 2xx error handling (Go analogue of Rust's SAFE206) | Flags `if err != nil { }` with an empty (or comment-only) body - the silently swallowed error. The condition shape to match: a binary `!=`/`==` comparison where one operand's identifier text is `err` (configurable `error_names_go`, default `["err"]`) and the other is `nil`. Disabled by default. |
| SAFE211 | `panic_calls_outside_tests` | 2xx (Go analogue of Rust's SAFE204) | Flags `panic(...)` calls outside `_test.go` files. Production paths should return errors. `log.Fatal*` / `os.Exit` optionally in the list (`panic_calls_go`, default `["panic"]`). Disabled by default. |

(SAFE210 was deliberately not proposed - keep the v1 Go-only set minimal; a
goroutine-leak or unchecked-type-assertion rule can follow with demand.)

## 3. Framework / runtime presets

None in v1. Record the future axis in the language page: a
`[tool.safelint.go] framework` preset (gin / echo / chi) would extend
`sources_go` with framework request accessors (`c.Param`, `c.Query`,
`c.PostForm`) and belongs to the Part B preset standard in
`docs/contributing/adding-a-language.md`; any Gin/Echo structural rules would
take the 9xx band.

## 4. Tests

Per the standard: per-rule `tests/rules/test_<rule>_go.py` (violation + clean
each), `tests/core/test_engine_go.py` (discovery, `// nosafe` suppression,
one known-bad file), optional-grammar hint test, allow-list bucket updates in
`tests/core/test_engine.py` (every widened tuple moves buckets), dataflow
tests for the SAFE801/802 lists, and the two new Go-only rules with
fires/clean/config-override cases each.
